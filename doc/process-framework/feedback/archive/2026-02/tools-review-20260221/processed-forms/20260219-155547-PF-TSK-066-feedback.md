## Improvement Suggestions

### What worked well
- New-FDD.ps1 and New-ArchitectureDecision.ps1 are excellent — reliable, auto-tracking, clear output
- Extraction-first principle saved time by reusing HOW_IT_WORKS.md content
- Master state session log provides good continuity across sessions
- Priority ordering (Foundation → Tier 3 → Tier 2) is practical

### What could be improved
- Two out of four file creation scripts had broken template paths
- Templates are Flutter/Dart-oriented, requiring significant adaptation for Python projects
- No validation that scripts work correctly before first use (no automated test)
- Three separate tracking updates per document (master state, feature tracking, id-registry) — scripts handle 2 of 3 but not master state

### Specific suggestions
1. **Fix all script template paths**: Audit remaining scripts (New-DebtItem, New-RefactoringPlan, etc.) for similar path issues
2. **Add `$projectRoot = Get-ProjectRoot` check**: All scripts should verify this variable is set before using it
3. **Create Python-adapted templates**: Fork the Flutter templates to remove irrelevant sections (Widget Tests, Dart references)
4. **Add script self-test**: A simple `-WhatIf` or `-Validate` flag that checks template paths exist without creating files
