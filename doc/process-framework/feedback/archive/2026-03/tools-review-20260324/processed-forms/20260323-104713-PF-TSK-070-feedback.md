## Improvement Suggestions

### What worked well
- Scripted test execution pipeline is fast and reliable for standard test cases
- Orchestrator handles LinkWatcher lifecycle (start/stop/restart) cleanly
- Bug reporting and tracking update scripts integrate seamlessly
- 15 out of 17 test cases ran successfully on first attempt

### What could be improved
- Run-E2EAcceptanceTest.ps1 needs a way to pass per-test-case LW flags (e.g., `--dry-run`, custom `--project-root`)
- Update-TestExecutionStatus.ps1 group-level updates should not overwrite individual case notes when cases have mixed results
- Run-E2EAcceptanceTest.ps1 should call Update-TestExecutionStatus.ps1 automatically after each test case (or as a batch at the end), eliminating the manual tracking update step
- Task definition step 2 mentions "install code changes globally" but the AI agent skipped this because it's easy to miss — the step should be more prominent or the orchestrator script should verify/warn about installation freshness

### Specific suggestions
- Add `-LinkWatcherArgs` parameter to Run-E2EAcceptanceTest.ps1 that reads from test-case.md frontmatter or a config field
- When Update-TestExecutionStatus.ps1 applies a group status, preserve notes for cases that already have a different status
- Integrate Update-TestExecutionStatus.ps1 calls into Run-E2EAcceptanceTest.ps1 so test results are automatically recorded in test-tracking.md and feature-tracking.md as part of the test run
- Consider adding an installation freshness check to Run-E2EAcceptanceTest.ps1 (e.g., compare source file timestamps with installed files)
