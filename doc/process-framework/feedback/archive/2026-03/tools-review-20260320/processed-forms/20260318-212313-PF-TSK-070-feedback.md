## Improvement Suggestions

### What worked well
- Run-E2EAcceptanceTest.ps1 pipeline (Setup → run.ps1 → wait → Verify) is clean and effective
- Update-TestExecutionStatus.ps1 correctly updated tracking for features and workflows
- Verify-TestResult.ps1 with -Detailed gives clear line-by-line diff output on failures

### What could be improved
- Default WaitSeconds (5s) is too short for freshly set up workspaces. LinkWatcher needs to scan new files before the move action fires.
- The task definition should mention that code changes require global installation before E2E testing
- Run-E2EAcceptanceTest.ps1 could add a pre-run delay between setup and execution

### Specific suggestions
- Add a `-PreRunDelay` parameter to Run-E2EAcceptanceTest.ps1 (default 10-15s) that waits between setup completion and run.ps1 execution, giving LinkWatcher time to scan new workspace files
- Add a note in PF-TSK-070 Step 2: "If validating code changes, install the current version globally first (e.g., `python deployment/install_global.py`) and restart LinkWatcher"
