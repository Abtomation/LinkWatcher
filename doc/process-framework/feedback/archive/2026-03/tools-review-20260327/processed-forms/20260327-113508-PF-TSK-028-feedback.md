The process framework has a gap in when ADRs should be created. Currently ADR creation is only referenced during early design phases (System Architecture Review, Foundation Feature Implementation, TDD Creation). There is no systematic trigger for creating ADRs later in the lifecycle — e.g., when validation reports identify undocumented architectural decisions (as TD063 was discovered via PD-VAL-046), or when implementation reveals non-obvious design choices. This gap means architectural decisions accumulate as undocumented tech debt until validation catches them retroactively. Needs evaluation to determine where additional ADR creation triggers should exist in the framework.
