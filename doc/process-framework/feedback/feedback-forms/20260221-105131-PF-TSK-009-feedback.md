## Improvement Suggestions

### What worked well
- The preparation phase forced a complete audit rather than just fixing the two named scripts, discovering additional bugs
- The planning checkpoint with multiple approaches gave the user meaningful control
- The WhatIf validation pattern allowed quick verification without consuming IDs
- The process-improvement-tracking.md provided a clean source of truth for scope

### What could be improved
- DocumentManagement.psm1 should document its `[regex]::Escape()` behavior prominently so script authors know to use literal brackets
- The script-development-quick-reference.md already has a "Template Replacements Not Working" section but it could be more prominent or linked from the module itself
- Consider adding a validation step in `New-StandardProjectDocument` that warns if replacement keys contain backslash-escaped brackets

### Specific suggestions
1. Add a comment block at the top of `New-ProjectDocumentWithMetadata`'s replacement loop explaining that keys are auto-escaped — use literal brackets, not escaped ones
2. Consider adding a runtime warning: `if ($key -match '\\[') { Write-Warning "Replacement key '$key' contains escaped brackets — use literal brackets instead" }`
