## Improvement Suggestions

### What worked well
- Context/Decision/Consequences/Alternatives ADR structure maps naturally to retrospective documentation
- Extraction-First Principle prevents redundant content creation and ensures attribution
- FDD script's auto-update of feature tracking status reduces manual coordination
- Retrospective header pattern (with `retrospective: true` frontmatter) clearly marks documents as reverse-engineered
- Priority ordering (Foundation → Tier 3 → Tier 2) produced the right documentation sequence

### What could be improved
- Master state discrepancy detection: the ✅/⬜ state in master state is not validated against actual file existence — a 3.1.1 FDD was marked ✅ despite no file existing
- Documentation Requirements Summary totals in master state require manual calculation — no auto-aggregation from feature rows
- TDD tier validation: -Tier parameter is not validated against feature's assessed tier; wrong value silently creates wrong-depth document
- Guide files contain content for multiple use cases (new feature, retrospective, API-focused) without clear navigation — increases reading time before scripting
- FDD template contains non-Python language examples that create noise for Python projects

### Specific suggestions
1. Add a validation script (similar to `Validate-FeedbackForms.ps1`) that checks master state ✅ entries against actual file existence on disk — would have caught the 3.1.1 discrepancy automatically
2. Make Documentation Requirements Summary auto-compute from feature row data (or add a note that it requires manual update when rows change)
3. Add a "Retrospective Quick Reference" table to each guide listing only the parameters and steps applicable to retrospective use — skip the "new feature" guidance sections
4. Consider a `New-RetrospectiveDocSet.ps1` wrapper that runs FDD + TDD creation for a given Feature ID with one command, auto-updating all tracking files

## Additional Context

The largest challenge was the two-surface tracking system (master state + feature tracking) requiring separate manual updates after each document creation. ### Integration with other tools

The guides (PF-GDE-039, PF-GDE-029, PF-GDE-033) are consistent in their retrospective documentation approach, which aids smooth transitions between tool types within the same session.
