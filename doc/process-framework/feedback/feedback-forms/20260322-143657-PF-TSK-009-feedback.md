## Improvement Suggestions

### What worked well
- Batching 6 related IMPs in one session gave coherent results — each built on the previous
- User-driven scope decisions prevented over-engineering (fixture registry kept minimal, coverage kept project-level)
- Checkpoint-before-implement pattern caught the audit column mismatch before it caused more damage
- The New-ProcessImprovement.ps1 → work → Update-ProcessImprovement.ps1 lifecycle is smooth

### What could be improved
- Update-MarkdownTable silently ignoring unmatched column names caused data corruption that went undetected for weeks
- Python PATH not available in PowerShell required an auto-resolution workaround in Run-Tests.ps1
- Accidentally overwrote uncommitted E2E tracking data when restoring test-tracking.md from HEAD

### Specific suggestions
- Add a `-WarnOnUnmatchedColumns` switch (or make it default) to Update-MarkdownTable so column name mismatches are visible
- Consider adding a pre-run validation to Run-Tests.ps1 that checks python availability before test execution

## Additional Context

### Task-specific challenges
- PowerShell inherits Unix-format PATH from bash shell, causing python resolution failures
- test-tracking.md had uncommitted changes from a previous session that were destroyed during restoration — reinforces the CLAUDE.md prohibition on destructive working tree operations

### Integration with other tools
- Run-Tests.ps1 now integrates with TestTracking.psm1 (per-file tracking) and test-tracking.md (coverage summary)
- python-config.json and language-config-template.json gained testSetup section for framework portability
- pytest.ini and python-config.json now have cross-reference comments documenting their relationship
