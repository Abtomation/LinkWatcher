## 1. Feature Overview

### Feature Description

Provides reusable test helper classes, builder patterns, and assertion utilities in `tests/utils.py` and `tests/conftest.py`. The `tests/utils.py` module contains: `TestProjectBuilder` (fluent builder for creating test project structures with markdown, YAML, JSON, and Python files), `LinkReferenceBuilder` (builder for constructing `LinkReference` objects), `PerformanceTimer` (context manager for timing operations), `MockFileSystemEvent` (mock for watchdog events), and utility functions (`create_sample_project`, `create_large_project`, `temporary_project`, `simulate_file_move`, `wait_for_processing`, `generate_*` data generators, assertion helpers).

The `tests/conftest.py` provides `TestFileHelper` (static methods: `create_markdown_file`, `create_yaml_file`, `create_json_file`) exposed as the `file_helper` fixture, plus custom assertions `assert_reference_found` and `assert_reference_not_found` added to the pytest namespace. Together these utilities eliminate boilerplate across all test categories.

### Business Value

- **User Need**: DRY test code â€” complex test setups reduced to single builder calls
- **Business Goal**: Lower the barrier to writing new tests; consistent project structure creation
- **Success Metrics**: All test categories use shared utilities; no duplicated project-creation code

### Scope

**In Scope**:

- `tests/utils.py` â€” `TestProjectBuilder`, `LinkReferenceBuilder`, `PerformanceTimer`, `MockFileSystemEvent`
- `tests/utils.py` â€” Utility functions: `create_sample_project()`, `create_large_project()`, `temporary_project()`, `simulate_file_move()`, `generate_*()` data generators
- `tests/utils.py` â€” Assertion helpers: `assert_references_equal()`, `assert_file_contains()`, `assert_file_not_contains()`
- `tests/conftest.py` â€” `TestFileHelper` class and `file_helper` fixture
- `tests/conftest.py` â€” Custom pytest assertions: `assert_reference_found`, `assert_reference_not_found`

**Out of Scope**:

- pytest fixture definitions (covered by 4.1.1)
- Static test data files (covered by 4.1.6)

## 7. Design Decisions

### Decision 1: Builder Pattern for Test Data Construction

**Context**: Tests need to construct complex project structures and link reference objects with varying configurations

**Decision Made**: Use fluent builder pattern (`TestProjectBuilder`, `LinkReferenceBuilder`) with chained method calls returning `self`

**Rationale**: Builder pattern allows tests to specify only the fields they care about while getting sensible defaults for the rest. Fluent API reads naturally: `builder.add_markdown_file("a.md").add_yaml_file("b.yaml").build()`

**Implications**:

- New file types can be added by extending the builder with new `add_*` methods
- Tests remain concise even for complex project structures
- Builder handles all temp directory management internally

### Decision 2: Context Manager for Temporary Projects

**Context**: Test projects need automatic cleanup to prevent temp directory leaks

**Decision Made**: `temporary_project()` is a context manager (`@contextmanager`) that creates and yields a temp project, then removes it on exit

**Rationale**: Context manager pattern guarantees cleanup even on test failure. Pairs naturally with pytest's fixture teardown model.

**Implications**:

- No manual cleanup needed in tests
- Works with both `with` statements and pytest fixtures

### Decision 3: Separate Assertion Helpers

**Context**: Tests frequently check file contents and reference equality

**Decision Made**: Provide standalone assertion functions (`assert_references_equal()`, `assert_file_contains()`, `assert_file_not_contains()`) in `tests/utils.py`, plus custom pytest namespace assertions in `tests/conftest.py`

**Rationale**: Two complementary assertion approaches â€” `utils.py` helpers for general use, `conftest.py` namespace assertions (`pytest.assert_reference_found`) for pytest-idiomatic access. Both provide better error messages than raw `assert`.

**Implications**:

- Assertion failures include descriptive context (expected vs actual references)
- Consistent assertion API across all test categories
