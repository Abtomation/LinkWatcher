- Step 1's "independently verify tech debt" mandate — surfaced a real production bug that would have gone undetected if I'd taken TD213's "by design" framing at face value.

### What could be improved
- **Update-TechDebt.ps1 -PlanLink overwrites the ID column** (already tracked as PF-IMP-620; not a new finding, but the per-session manual repair cost is real — consider raising priority).
- **New-BugReport.ps1 hidden constraints**: `-DiscoveredBy` ValidateSet and `-Component` 50-char limit aren't shown in `-?`; both caused retry cycles.
- **New-RefactoringPlan.ps1 batch mode**: when `-DebtItemId` is comma-separated, the script could generate N item sections instead of one (currently I copy/paste the second block manually).

### Specific suggestions
1. PF-IMP-620 fix priority: bump from current backlog to Active. Affects every -PlanLink invocation.
2. New-BugReport.ps1: surface ValidateSet values for `-DiscoveredBy` in `-?` help text; surface character-length constraints for `-Component`/`-Title`.
3. New-RefactoringPlan.ps1: when `-DebtItemId` contains comma, generate one Item section per ID with the IDs prefilled.
