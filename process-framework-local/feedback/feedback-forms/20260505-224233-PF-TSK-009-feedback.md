## Improvement Suggestions

### What worked well
- Step 3 evaluation criteria caught both rejection cases (IMP-746 first pass; IMP-742 after human pushback). The Recurring Value + Minimum Viability + Framework Fit triangle is the discriminating combination for single-occurrence callout IMPs.
- Memory rules (No Anti-Pattern Callouts From Single Occurrences, Apply IMP Minimum Viability Rigorously) acted as a fast-path filter — both IMPs trip the same memory pattern.
- Step 6 checkpoint format (Problem Summary → table → options) gave the human partner concrete artifacts to challenge. The "options for your decision" framing made it natural for the human to push back ("I think this gap should already be filled") rather than rubber-stamp.
- Counter-proposal evaluation directive (IMP-731) triggered correctly on the IMP-742 re-evaluation — when my Option A was challenged, I ran my own counter-proposal (rejection) through the same Step 3 criteria and found it more defensible than my original recommendation.

### What could be improved
- IMP-742 first-pass evaluation rated Recurring Value as Medium based on PF-PRO-030's institutionalization of pilot lifecycles, but missed that PF-PRO-030 explicitly designed pilot resolution as **terminal** (Resolved → archive). This is a Framework Fit gap I should have caught — codifying a "revise resolved concept" callout normalizes decision reversals contrary to design intent. The check I missed: when evaluating a "fill the gap in the documented flow" IMP, also check whether the documented flow was *intentionally* one-directional.
