## Improvement Suggestions

### What worked well
- The Effort Assessment Gate's "verify the description independently" framing — it's the right scrutiny even when the description turns out accurate, because it forces reading the actual branch/loop structure.
- L8 test-only shortcut. Saved 7 grep-and-justify exercises that would all have come back N/A.
- L7 baseline-vs-final diff approach. Made "0 owned regressions" verifiable instead of asserted.
- Audit-flagged TD closure logic at L10 — the explicit "if ALL findings closed → run X; if partial → re-audit" branching kept me from incorrectly auto-flipping audit status.
- New-RefactoringPlan.ps1, Update-TechDebt.ps1, New-FeedbackForm.ps1: all three produced clean, well-scoped artifacts with no manual restructuring needed.

### What could be improved
- Run-Tests.ps1 swallows pytest output. The wrapper banner says "✅ All tests completed successfully!" even when 1 test failed, and the actual pytest summary (counts and FAILED test names) is absent. This makes it unusable for the L3 baseline step which explicitly requires the failure name.
- Validate-TestTracking.ps1 doesn't check the test-tracking.md count column against pytest collection — only marker-vs-collection. So adding a test method silently leaves the count stale until human review.

### Specific suggestions
- **Run-Tests.ps1**: Forward pytest stdout/stderr to the console (`tee` or unbuffered redirect), and base the success banner on pytest's exit code (non-zero → red, with summary line and FAILED names visible). Or accept a `-PassThruOutput` switch that skips the wrapper's quiet mode for L3-style baseline use.
- **Validate-TestTracking.ps1**: Add Check 8: tracking-row "Test Cases" / "Last Run" count vs pytest collection count, with `WARNING` on drift. Would have surfaced the 5→6 drift in test_shouldmonitorfileancestorpath.py automatically.
