## Improvement Suggestions

### What worked well
- **Synthetic-harness verification** (Step 10 medium-risk path): running the fixed script with `-WhatIf` immediately surfaced the second-order blocker (`directoryStructure` missing) before any production-like invocation.
- **Step 6 minimum-viability gate** triggered correctly for PF-IMP-750, leading to the MVP carve-out (skipping ~6 contextually-fine sites where the bare path matched the file's internal abbreviation convention).
- **"Observe before broadening" memory rule** applied naturally: PF-IMP-749 fixed only the IMP-stated bug and surfaced the parallel `Validate-StateTracking.ps1:1472` bug as observation; user approval expanded scope explicitly rather than the agent extending unilaterally.
- **`feedback_db.py log-change --batch`** with stdin JSON handled 14 changes in one call — much cleaner than 14 separate invocations.

### What could be improved
- **`feedback_db.py log-change` UX for first-time tool registration**: Step 12 in PF-TSK-009 gives the `--new-tool` instruction but doesn't flag that **multi-tool batches are an all-or-nothing transaction** — a single unregistered tool aborts the whole batch with no partial commit. The fix is to add `--new-tool` upfront when the batch includes any guide/template/registry that's never been a feedback target before. Suggest noting in Step 12 that `--new-tool` is a safe default for batches where some entries may be first-time registrations.
- **PF-IMP-749's description under-stated the scope**: the IMP described two coupled fixes (script + doc) but the actual remediation required three (script + missing config section + doc). The IMP source-of-truth should ideally be revisable when scope expands — currently the expansion is captured in completion notes only. This is probably acceptable since the completion notes are searchable, but it does mean the IMP description in archive doesn't reflect what was actually done.

### Specific suggestions
- **PF-TSK-009 Step 12**: Add a one-liner: "For batches that may include first-time-registered guides/templates/registries (anything outside the script/task ID conventions), pass `--new-tool` upfront — the batch is all-or-nothing and aborts on the first unknown ID."
