## Improvement Suggestions

### What worked well
- Multi-session state file with cross-session findings section — later sessions build on earlier insights
- Parallel session execution — 4 sessions running concurrently significantly reduces total wall-clock time
- Enumeration requirement forces precision in artifact counts
- Industry research step provides external calibration that prevents insular evaluation

### What could be improved
- IMP IDs in the report need post-registration correction when parallel sessions consume IDs. A batch registration mode for New-ProcessImprovement.ps1 could help.
- The evaluation report template could include an "Industry Research Context" section by default rather than requiring manual addition.

### Specific suggestions
- Add `-BatchFile` parameter to New-ProcessImprovement.ps1 that accepts a JSON/CSV file of multiple improvements to register in one command
- Add "Industry Research Context" section to framework-evaluation-report-template.md
