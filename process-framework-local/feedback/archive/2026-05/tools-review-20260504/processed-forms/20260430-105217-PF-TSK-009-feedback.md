## Additional Context

### Task-specific challenges
- **Bash + PowerShell quoting for regex with both `"` and `'`** (IMP-684 Update-LanguageConfig.ps1 invocation): the `featureMarkerPattern` regex `pytest\.mark\.feature\(["']([^"']+)["']\)` contains both quote types, making single-line bash → pwsh invocation hard. Solved with the temp-file pattern from CLAUDE.md script-development guide. No friction beyond the initial detour.
- **Update-LanguageConfig.ps1 empty-default rejection** (IMP-684): `[string]$DefaultValue` was mandatory and rejected empty strings, forcing a 1-line `[AllowEmptyString()]` side-fix before the field could be added with empty default. Surfaced as an in-session decision to the human partner before applying.
- **Smoke-testing without side effects** (IMP-638): the script's early `ShouldProcess` return on `-WhatIf` blocked using -WhatIf to verify the multi-row count. Worked around by extracting the loop logic into a standalone test script — exercised the exact logic with zero file mutations or ID-registry consumption. Same pattern reused for the precision test (Tests 5 and 6).

### Integration with other tools
- **PF-TSK-009 task definition** orchestrates `Update-ProcessImprovement.ps1` (state mutation), `feedback_db.py log-change` (trend tracking), and `New-FeedbackForm.ps1` (session-end checklist). The integration is seamless when each tool's specific friction (-Confirm:$false plumbing, bash-quoting hints, --new-tool flag) is documented in the task definition or its companion guides.
- **Update-LanguageConfig.ps1 + project-config.json + language config files** form a clean schema-extension pattern that worked exactly as designed once the empty-default issue was fixed. The `Get-LanguageConfig` helper (in Common-ScriptHelpers Core.psm1) and the existing `Run-Tests.ps1` consumer pattern provided a clean template for the new consumer in `New-RefactoringPlan.ps1`.
