## Improvement Suggestions

### What worked well
- Step 2 evaluation table is the load-bearing element of PF-TSK-009 — Minimum Viability + Data-Driven Validation rows turned IMP-631 from "implement as written" into "reject as subsumed by PF-PRO-028".
- Active pushback (memory note `feedback_active_pushback.md`): when the user surfaced that New-Handbook.ps1 already has self-verification, the right move was to investigate honestly and report 96% adoption gap rather than pivot to a smaller framing. Worked.
- The PF-IMP-659 fix could be dogfooded immediately (used the new .NOTES bullet to keep PF-IMP-687's Description under 500 chars on first try). Direct evidence the fix solved the original friction.
- Single edit pass for IMP-659 + soak adoption avoided two hash transitions (each would have reset the counter). Combining them gave a clean demo trajectory.
- The user-approved exception to the 3-IMP session limit was justified: cycle #4 (IMP-659 with soak demo) was meaningful and quality didn't visibly degrade, but I noticed the discipline cost in form-filling (now). Future session-limit exceptions should still finalize at the same level.

### What could be improved
- Register-SoakScript silent success is friction (see Tool 4 Clarity comment). Suggestion: emit one-line confirmation at default verbosity.
- I missed that PF-IMP-627 was already resolved when proposing it as a test target — should have grepped for status before suggesting. The user caught it. (Filed observation: when listing candidate IMPs for any work, always verify Status column before recommending.)
- The IMP-685/686 filing exposed the 500-char limit in real time — that retry could have been avoided had the constraint been visible at parameter-help level (which is exactly what IMP-659 fixed). The session demonstrates the problem and the fix in one arc.
- The closeout-decision-translation gap (IMP-686's premise) is real and broader than PF-TSK-026. The framework relies on individual session discipline to translate "deferred decisions" from archived temp state files into active tracking artifacts, and that discipline was absent in PF-TSK-026 Phase 5.
