## Improvement Suggestions

### What worked well
- Step 2's "IMP is raw input, not specification" framing caught the IMP-668 mislabeling (IMP said "L8" but actual fix belongs at L3/L7). Without that gate, the agent would have applied the change at the wrong location.
- Step 3's Minimum Viability gate forced honest comparison on IMP-678 — rated Recurring Value Low and Complexity-to-Benefit Marginal, which surfaced rejection as a real option rather than ceremonial.
- The Step 6 "do not enumerate ideas you discarded during exploration" rule kept the option presentations focused.
- The Conciseness rule at Step 3 (one-line summary when all favorable) saved real time on IMP-664 — full table only appeared for IMP-668 and IMP-678.

### What could be improved
- IMP descriptions occasionally reference artifact IDs that don't resolve (IMP-678 said "PF-TSK-048" but PF-TSK-048 has never been registered; actual task is PF-TSK-026). The Step 2 "Verify the problem" prose doesn't currently include "verify all artifact references resolve" as an explicit sub-step. An agent reading the IMP literally could conclude the IMP is invalid because the named artifact doesn't exist, when in fact the IMP is valid and the description is mislabeled.
- Step 13's checkpoint sometimes restates what was already shown at Step 6. For low-risk content edits, the diff shown at Step 13 is mostly identical to the proposal shown at Step 6.

### Specific suggestions
- At Step 2 "Verify the problem", add a sub-step: *"Verify that artifact IDs and file paths in the IMP description resolve to real artifacts (`PF-id-registry.json`, `PD-id-registry.json`, file existence). If a reference doesn't resolve, treat it as a labeling error in the IMP — locate the actual artifact via search before evaluating, do not reject the IMP solely because a referenced ID is wrong."*
- At Step 13, allow a compressed checkpoint format for low-risk changes that were fully described at Step 6: *"Risk class confirmed: <class>. No deviation from approved Step 6 plan. Approve / revise / reject?"* — avoids restating the diff verbatim.
