## Improvement Suggestions

### What worked well
- Single-command per-group invocation drove the whole pipeline
- Auto-tracking eliminated manual e2e-test-tracking.md edits across 22 cases
- Group rollup (case statuses → group status) computed correctly
- Workflow milestone status auto-recalculated
- Clear pass/fail summary at the end of each group

### What could be improved
- Stray LinkWatcher cleanup between cases when `skip_lw_start` mode is used (TE-E2E-015 setup failure)
- "Scan wait timed out after 30s" appears on every run — either the timeout is too tight for this project size, or the readiness signal is unreliable
- Update-TestExecutionStatus.ps1 should be consistent about whether it clears the Notes column on rollup (case-level cleared, group-level not)

### Specific suggestions
1. Add a `Stop-WorkspaceLinkWatcher` step in the orchestrator's per-case loop, run before next case's Setup, that kills any python process whose CWD or argv references the workspace path.
2. Investigate the scan-wait timeout: either bump the default to 60s for projects >5k files, or fix the scan-complete signal so it actually fires before timing out.
3. Decide whether group-level Notes get cleared on rollup-to-Passed and apply consistently.
