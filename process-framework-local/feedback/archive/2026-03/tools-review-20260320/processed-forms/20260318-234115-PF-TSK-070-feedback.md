## Improvement Suggestions

### What worked well
- Stop/start LW lifecycle per test case eliminates event noise completely
- Log-based scan completion detection (polling for `completed_initial_scan`) is robust and avoids hardcoded wait times
- Line ending normalization in Verify-TestResult.ps1 prevents false failures

### What could be improved
- Initial scan time (~40-65s per test) dominates execution time
- Run-E2EAcceptanceTest.ps1 sets up ALL test cases in the group even when running just one (`-TestCase`)

### Specific suggestions
- Consider scoping LW to only the workspace subdirectory during E2E tests to reduce scan time
- Setup-TestEnvironment could accept a `-TestCase` parameter to only set up the relevant test case directory

## Additional Context

### Task-specific challenges
Two infrastructure issues discovered and fixed during execution:
1. LW's dir_move_detector correlates delete+create events across test case directories during cleanup/setup — solved by stopping LW before setup
2. Initial scan takes ~40s on this project — solved by polling the log for `completed_initial_scan` instead of using a hardcoded wait
3. Verify-TestResult.ps1 raw comparison was sensitive to CRLF vs LF and trailing newlines — solved by normalizing both sides

### Integration with other tools
The test case creation task (PF-TSK-069) produced well-structured fixtures that worked correctly once the infrastructure issues were resolved.
