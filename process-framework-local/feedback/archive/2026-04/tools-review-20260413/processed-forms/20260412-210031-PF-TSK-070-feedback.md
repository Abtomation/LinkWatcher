## Additional Context

### Task-specific challenges
PD-BUG-087 blocked all 3 WF-009 test cases: `should_monitor_file()` in `utils.py:67-70` checks absolute path parts against ignored_directories, including ancestors above `--project-root`. The `e2e-acceptance-testing` ancestor directory matched `validation_extra_ignored_dirs`, causing 0 files scanned. This is the first time validation-mode E2E tests were attempted — watcher-mode tests don't use `should_monitor_file` in the same way.

### Integration with other tools
Run-E2EAcceptanceTest.ps1 couldn't be used (PF-IMP-494 — non-zero exit code handling). Manual execution path was straightforward but requires understanding which steps the orchestrator normally handles.
