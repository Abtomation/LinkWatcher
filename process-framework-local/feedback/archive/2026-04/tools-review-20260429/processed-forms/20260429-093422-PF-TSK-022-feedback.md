## Improvement Suggestions

### What worked well
- L8 Documentation & State Updates checklist: the explicit Integration Narrative line item caught PD-INT-009's stale description of the buggy behavior. High-leverage prompt.
- Batch mode in lightweight plans (TD232 + TD233 in one plan, one session) — combining related TDs fits the human partner's "combine if related" intuition.
- L5 checkpoint placement prevented over-investing in a design that hadn't been validated.
- L3 baseline + L7 diff anchor: turned "no regressions" from a claim into a verifiable diff.
- Discovery of the abandoned PD-REF-199 plan (yesterday's identical work) — handled cleanly by archiving it with a "superseded by PD-REF-210" note.

### What could be improved
- **Run-Tests.ps1 -All output hides pytest summary line** (counts of passed/failed/skipped). L3 explicitly requires those numbers; agents must run pytest directly as a workaround.
- **New-TestFile.ps1 generates C-style block-comment header `/* ... */`** in `.py` files, which is invalid Python syntax. Forces overwrite-from-scratch rather than incremental fill.
- **Lightweight plan template doesn't accommodate batch items that share one fix shape**. When two TDs are resolved by extracting a single shared helper, neither item fully owns the helper extraction. I added a non-template "Common Approach" section to handle this.

### Specific suggestions
- Run-Tests.ps1: print or propagate the last pytest summary line (e.g., `tail -1` of the pytest stdout, or a parsed `{passed, failed, skipped, xfailed}` line). Could be triggered by a `-Verbose` flag if conciseness is a concern by default.
- Python test file template: replace the `/* ... */` block-comment header with a Python module docstring (`"""..."""`) carrying the same metadata fields.
- Lightweight refactoring plan template: add an optional commented-out "Common Approach (applies to all items)" section between Items, with a brief instruction that batch users with shared fixes should keep it. Documented in the L1 step or in the lightweight path's batch-mode note.
