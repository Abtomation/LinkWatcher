## Improvement Suggestions

### What worked well
- Step 1 effort gate's "independently verify TD descriptions" warning caught the line-109 scope expansion before commitment. Without it I would have either silently rewritten 22 tests or shipped half of TD242.
- L3 baseline + L7 diff produced a binary signal (clean / 22 broken / clean again) that made the scope-correction decision easy to surface to the human partner.
- New-RefactoringPlan.ps1 + Update-TechDebt.ps1 chained cleanly with the PD-REF-207 ID propagating through all artifacts.

### What could be improved
- L4 could mention transitive call sites: tests that hit a refactored branch via higher-level APIs (e.g., update_references() → _calculate_new_target() → branch under test) won't show up by grepping for the changed method name.
- The Item 1 header in the lightweight plan template (`[Debt Item ID] — [Brief Description]`) could be auto-populated when New-RefactoringPlan.ps1 is called with `-DebtItemId`.

### Specific suggestions
- Add a sub-bullet at L4: *"For tests, also identify transitive call sites — tests that exercise the affected branch through public APIs without naming the internal method."*
- New-RefactoringPlan.ps1: when `-DebtItemId` is provided, fetch the description from technical-debt-tracking.md and substitute into the Item 1 header. (Optional — current behavior is fine, this would just save a re-type.)
- Consider an "Internal-helper-only" batched-N/A shortcut at L8 for refactors that add or extract a private helper with no public surface change. Same shape as the existing Tier 1 / Test-only / Documentation-only shortcuts.

## Additional Context

### Task-specific challenges
TD242's description proposed two changes — helper extraction (consolidation) and a new line-109 guard — coupled in one debt item. The helper extraction was a safe pure refactor; the line-109 guard was a behavior change with non-trivial test churn. The Step 1 effort gate's "verify independently" warning surfaced this coupling explicitly. After investigation, the human chose to ship only the consolidation portion; the line-109 portion was deferred with rationale documented. This is a healthy outcome but suggests TD descriptions that bundle "do X, and also do Y" should be flagged at intake — Y might be a separate TD if it has independent risk profile.
