## Improvement Suggestions

### What worked well
- Audit-flagged TD with concrete evidence (PH-001 Run 1 16.01s vs Run 2 9.58s) and a named precedent (TD216 / PD-REF-196) made effort assessment fast.
- Test-only shortcut at L8 collapsed 7 doc-update line items into a single batched N/A.
- LinkWatcher auto-rewrote the `-PlanLink` argument when the plan was moved to archive, saving a manual edit.
- L3-baseline / L7-diff structure made the +4.81s wall-time delta cleanly attributable to warmup overhead and turned "did I break something?" into a one-line comparison.

### What could be improved
- Lightweight-path L3 prescribes `Run-Tests.ps1 -All` but `-All` excludes slow tests on this project, which produces an incomplete baseline for refactorings targeting slow tests. Workaround: run `python -m pytest test/automated/` directly. Doc could surface this scenario.
- `New-RefactoringPlan.ps1` emits all 7 doc-update line items even when `-TargetArea` clearly points to a test directory; adding a `-TestOnly` switch (or auto-detection) would let the script pre-collapse them.

### Specific suggestions
- Lightweight-path L3: add a one-line note like *"If the refactoring targets slow tests, use `python -m pytest test/automated/` directly — `Run-Tests.ps1 -All` excludes slow."*
- `New-RefactoringPlan.ps1`: detect `-TargetArea` matching `test/**` or `test_*.py` and auto-emit the test-only shortcut block instead of the full 7-item list.
