## Improvement Suggestions

### What worked well
- Step 15 "write tests first; verify they FAIL for the right reason" — reproducing the documented corruption signature character-for-character was the strongest possible confirmation of the root cause hypothesis before writing any fix code.
- The combined Step 11/14 checkpoint for M-scope bugs eliminates redundant approvals.
- Update-TestExecutionStatus.ps1 cascading through cross-cutting tests is exactly right — manual maintenance would be error-prone.

### What could be improved
- Update-BugStatus.ps1 has no `-RelatedFeature` parameter. When triage corrects the Related Feature column (as in this case, 1.1.1 → 2.2.1), the agent must edit the row manually after the script runs.
- Bash quoting friction with `pwsh.exe -File` when triage/fix notes contain backticks. The argument arrives at PowerShell stripped, with no warning.

### Specific suggestions
- Add `-RelatedFeature` parameter to Update-BugStatus.ps1 (matches the existing -Workflows pattern).
- For arguments containing `$`, backticks, or other shell-special chars, consider documenting the temp-file fallback pattern more prominently in the script's `-?` help, or accept a `-NotesFile` parameter that reads the field from a file (sidestepping shell quoting entirely).
