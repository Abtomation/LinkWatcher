## Improvement Suggestions

### What worked well
- **L1 independent verification guidance**: The "verify TD descriptions" instruction prevented me from blindly applying audit's stale tolerance numbers. Concrete payoff: discovered 4-13× drift before code changes were locked in.
- **L8 test-only documentation shortcut**: Saved ~30 redundant N/A justifications across 5 items × 7 doc types.
- **L10 audit-loop closure logic**: The "do NOT mark Audit Approved if findings partially addressed" rule applied directly here (TD219/240 still open) — clean guardrail.
- **Update-TechDebt.ps1 reliability**: Four sequential closures, zero issues, clear output, accurate state-file mutations.
- **Plan template's batch-mode hint**: The "Copy Item N section" comment in the Item 1 placeholder block was small but useful.

### What could be improved
- **Run-Tests.ps1 output suppression**: Hides pytest summary (pass/fail counts), making it unusable for L3 baseline capture. Forced fallback to direct pytest.
- **No path guidance for stale-audit-recommendations scenario**: When tolerance values from an older audit are calibrated to baselines that have since drifted, the AI has to invent a calibration approach mid-session. The path could include explicit guidance.
- **New-RefactoringPlan.ps1 mode flag mutual exclusivity**: Doesn't accommodate hybrid cases (e.g., lightweight refactoring of performance test code).
- **Plan template bundled-mode scaling**: Single Item 1 placeholder requires manual replication for batch sessions. A `-ItemCount N` parameter would scale.

### Specific suggestions
1. **Run-Tests.ps1**: Either pass through pytest's summary line (1-2 lines), or provide a `-Verbose` flag that does.
2. **Code Refactoring Lightweight Path**: Add to L1 (or new sub-step): "If audit-recommended numeric values were calibrated to baselines that have since drifted significantly (>2× from current measurements), recalibrate to current measurements at the same ratio. Document the recalibration in the plan; mark affected rows ⚠️ Needs Re-baseline; route to PF-TSK-085."
3. **New-RefactoringPlan.ps1**: Add `-ItemCount N` parameter that pre-generates N Item sections.
4. **New-RefactoringPlan.ps1**: Allow combining `-Lightweight` with `-Performance` (both apply meaningfully — lightweight=path complexity, performance=domain). Or document which to choose when both apply.
