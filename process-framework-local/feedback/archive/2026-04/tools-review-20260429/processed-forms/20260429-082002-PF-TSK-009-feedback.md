## Improvement Suggestions

### What worked well
- **PF-TSK-022 effort assessment gate caught a misclassification before any code work** — the user redirected from "tech debt" to "process improvement" within 2 turns. The Step 1 routing in PF-TSK-009 already supports IMP→TDI/BUG/FRQ reclassification; the inverse routing (TDI→IMP) used in this session is symmetric but undocumented.
- **PF-IMP-619 Step 5 minimum-viability gate** produced Option B (fail-fast only, ~5 lines) instead of the more complex Option A (fail-fast + disk uniqueness check). Avoided implementing speculative defenses against an unobserved failure mode.
- **PF-IMP-620 Step 5 active-pushback** caught that the IMP author's proposed fix (Option C — append PlanLink to Notes) abandoned a documented design feature. Settled on Option B (auto-wrap + reject malformed) which preserves the design while fixing the bug.
- **Per-IMP grep sweep at Step 11** confirmed no active docs described old behavior. Quick to do, valuable to verify.
- **Cleanup of 10 corrupted rows post-IMP** fit naturally into the session — natural follow-up to fixing the script that produced them.

### What could be improved
- **TD→IMP reclassification path is implicit, not documented.** PF-TSK-009 Step 5 has explicit IMP→TDI/BUG/FRQ routing; PF-TSK-022 Step 1 has Rejected workflow but doesn't surface the option of routing rejected TDs into IMP. The inverse routing is just as common and would benefit from the same explicit guidance.
- **No "row-cleanup" capability in `Update-TechDebt.ps1`.** Cleanup of the 10 corrupted Recently Resolved rows had to be done with manual Edit calls (one per row). For data-fix scenarios on resolved rows, a `Update-TechDebt.ps1 -RepairResolvedRow -DebtId "TDxxx" -ColumnName "ID" -NewValue "..."` (or similar) would scale better.

### Specific suggestions
1. **PF-TSK-022 Step 1 — add TD→IMP routing** to the Rejected workflow when the source-task analysis reveals the work is fundamentally a process-improvement (framework infrastructure modification) rather than product/test debt. Mirror the structure of PF-TSK-009 Step 5's IMP→TDI/BUG/FRQ routing, with the `process-framework/`, `doc/` = IMP heuristic.
2. **(Lower priority) Update-TechDebt.ps1 row-repair mode** — optional enhancement to handle data-fix scenarios on Recently Resolved rows without manual Edit calls. Would have shaved ~5 minutes off this session's cleanup.
