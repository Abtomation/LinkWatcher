## Improvement Suggestions

### What worked well
- The Step 1 "Independently verify tech debt descriptions" warning successfully caught the TD244 description discrepancy (claimed ~14 sites / 8 PH tests; actual 24 sites / 6 PH tests using time.time).
- L10's audit-partial-closure rule cleanly prevented an inappropriate "Audit Approved" mark on TE-TAR-070.
- `Update-TechDebt.ps1` MSYS-mangling detection and error message saved investigation time.
- Lightweight path's test-only N/A batching avoided 7 redundant N/A justifications.
- `New-BugReport.ps1` insertion into the correct priority section happened automatically — no manual table editing.

### What could be improved
- `Run-Tests.ps1 -All` output suppresses pytest's pass/fail/skip counts, showing only "✅ All tests completed successfully!". For L3 baseline + L7 regression diff, exact counts are essential and I had to fall back to running `pytest` directly. Wrapper should at least pass through the pytest summary line.
- `New-BugReport.ps1` ValidateSet members for `-DiscoveredBy` and `-Environment` are not visible in `-?` output — only revealed on rejection. Three sequential ValidateSet failures cost ~90s of round-trips. Surfacing these in the synopsis (or auto-listing them on `-?`) would be a one-line fix.
- `New-RefactoringPlan.ps1 -FeatureId` accepts any value without cross-checking against the test file's `pytest.mark.feature` marker. For test-file refactorings, auto-inferring feature_id (or warning on mismatch) would prevent the manual frontmatter correction I had to make.

### Specific suggestions
- Make `Run-Tests.ps1` always print the pytest tail summary (e.g., last line containing `passed|failed|skipped`) regardless of success — script-wrapping the suite shouldn't hide the result counts.
- Add a `-ListValidValues` flag (or always-include block in `-?`) to scripts with ValidateSet parameters, listing accepted values. Pattern: `Get-ValidValues -ParameterName "DiscoveredBy"`.
- Add an optional check in `New-RefactoringPlan.ps1` that, when `-TargetArea` points to a test file with a `pytest.mark.feature` marker, compares the marker value to `-FeatureId` and warns on mismatch.
