## Improvement Suggestions

### What worked well
- L7 ownership-analysis discipline forced a clear, defensible call on a pre-existing test failure that surfaced mid-session — preventing both false-attribution-as-regression and quiet dismissal.
- DA-category guidance at L2 distinguished initial coupling from drift, which sharpened the root cause analysis (single-commit `git log` showed the script was authored with concrete IDs from inception).
- Update-TechDebt.ps1's automatic Registry → Recently Resolved migration is one of the cleaner "close the loop" automations in the framework — single command, complete update.
- The plan's L11 checkpoint structure made it natural to surface a pre-existing test failure honestly, with full forensic context, rather than burying it.

### What could be improved
- L8 has shortcuts for Tier 1 / Test-only / Documentation-only refactorings, but no shortcut for **framework-infrastructure** refactorings — i.e., changes targeting `process-framework/` scripts, guides, tasks, or templates that have no LinkWatcher feature linkage. This pattern recurs (any time a framework script gains/loses a feature, or framework docs are tightened), and the current shortcuts force inline justification each time.
- L10/L11 ordering is awkward when L10's actions feel irreversible. The Lightweight Path tells you to update state files (TD resolution, plan archive) before checkpointing with the human partner, but if the human wants to redirect scope at L11, those state changes need to be undone.

### Specific suggestions
- Add a fourth shortcut to L8: **"Framework-infrastructure shortcut": If the refactoring targets exclusively framework artifacts (process-framework/ scripts, guides, tasks, templates, or context maps) with no impact on LinkWatcher feature code, batch items 1–7 as N/A with a single justification: "Framework-infrastructure refactoring — feature-level design and state documents do not reference framework artifacts." Still check item 8 (tech debt) individually.**
- Either swap L10 and L11 ordering in the Lightweight Path, or add a callout to L10 saying "If the changes in L10 would be non-trivial to reverse (e.g., archived plan, resolved TD), defer L10 until after the L11 checkpoint."
