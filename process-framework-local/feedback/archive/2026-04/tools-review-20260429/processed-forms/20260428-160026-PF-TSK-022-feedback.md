## Improvement Suggestions

### What worked well
- The Step 1 "Independently verify tech debt descriptions" callout is high-value — it directly led to confirming the integration narrative claim and discovering the cross-document update need (move from Orphan to Consumed table).
- L3 Test Baseline → L7 diff comparison made the new `test_logging_overhead` flake immediately classifiable as not-mine without ambiguity.
- The Lightweight Path document keeps everything in one file; no jumping between docs for procedure.

### What could be improved
- L10 doesn't address the test-tracking.md count-update convention when a refactoring resolution adds tests. The validator catches drift but the path document doesn't tell you the right way to fix it (Status column unchanged, count + notes updated).
- L8 item #6 ("Integration Narrative updated, or N/A") could include a hint for the common case of orphan-field resolutions: move the row from "Orphan configuration fields" table to the relevant "Consumed fields" table.

### Specific suggestions
- Add a one-line guidance under L10's `Validate-TestTracking.ps1` step: "When the validator reports a count mismatch on a file you added tests to, update the count and notes columns; leave the Status column unchanged unless the audit-coverage relationship has actually changed."
- Add a sub-bullet under L8 item #6: "If resolving an orphan-field TD: typically move the field from the narrative's 'Orphan' table to its 'Consumed' table and add a 'Resolved' pointer note above the orphan table referencing the refactoring plan."
