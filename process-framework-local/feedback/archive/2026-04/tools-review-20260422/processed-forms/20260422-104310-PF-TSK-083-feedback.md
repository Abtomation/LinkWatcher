## Improvement Suggestions

### What worked well
- Source-code-first verification culture: the explicit instruction to not trust TDDs and to verify against real code produced a materially stronger document — 7 concrete defects surfaced that would otherwise have stayed invisible.
- The two CHECKPOINTs (scope approval, completed-narrative review) gave natural pause points that matched how the work actually decomposes.
- The existing PD-INT-002 narrative was a far better style reference than the template scaffold alone — having a completed sibling made the expected depth and format unambiguous.
- Auto-updating `user-workflow-tracking.md`'s "Integration Doc" column is a small but high-value piece of automation that keeps cross-references in sync.

### What could be improved
- Task Step 7's `Update-TechDebt.ps1` example uses fictitious parameters (`-Title`, `-Category`, `-Severity`). Align with the real script.
- `New-IntegrationNarrative.ps1`'s docmap-insertion regex is brittle — matches the exact heading `### Integration Narratives` but this project's heading is `../../../../feedback-forms### technical/integration/ — Integration Narratives`. Loosen the match or make the failure hard.
- `Update-TechDebt.ps1 -Dims` accepts exactly one code, but the `-ListDims` output and the ValidateSet error format both imply multiple codes. Clarify or extend to accept multi-dim input.
- Narrative Template lacks a first-class "Alternate exit paths" sub-section, which WF-006 needed for the `--validate` branch.

### Specific suggestions
- Fix PF-TSK-083 Step 7 code sample to: `pwsh.exe -ExecutionPolicy Bypass -File process-framework/scripts/update/Update-TechDebt.ps1 -Add -Description "..." -Dims "CQ" -Location "..." -Priority "Medium" -EstimatedEffort "S" -Confirm:\$false`
- Loosen `New-IntegrationNarrative.ps1` docmap insertion to match on the H3 heading substring "Integration Narratives" (anywhere in the line) rather than the exact `### Integration Narratives` form. Alternatively, return a non-zero exit code when insertion fails so agents don't miss the warning.
- Add a `-BatchFile <json>` mode to `Update-TechDebt.ps1 -Add`, matching the pattern in `New-ProcessImprovement.ps1`.
- Either (a) extend `Update-TechDebt.ps1 -Dims` to accept `"EM,CQ"` or `"EM;CQ"` and split internally, or (b) update the `-ListDims` output to say "one code per invocation" so agents don't attempt multi-code calls.
- Add an optional "Alternate exit paths" section to the Integration Narrative Template for workflows with branching flows (`--validate`, dry-run mode, graceful-shutdown variants).
