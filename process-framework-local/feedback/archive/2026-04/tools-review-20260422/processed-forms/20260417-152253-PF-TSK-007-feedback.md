## Additional Context

### Task-specific challenges
The bug description attributed the root cause partly to "YAML frontmatter (--- delimiters) cause yaml.safe_load() ComposerError, falling back to GenericParser" — this is accurate for the YAML parser case but didn't cleanly explain the Markdown case (which never invoked YamlParser at all). The Markdown case has a separate root cause: MarkdownParser has no frontmatter-aware parsing, and its `bare_path_pattern` requires 3+ segments. The fix addresses both independent gaps.

### Integration with other tools
The fix integrates cleanly: MarkdownParser now uses YamlParser as a dependency for frontmatter, which is consistent with the already-existing YamlParser → GenericParser fallback pattern.
