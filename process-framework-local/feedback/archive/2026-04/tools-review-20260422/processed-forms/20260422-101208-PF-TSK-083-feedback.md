## Improvement Suggestions

### What worked well
- Source-code verification requirement is load-bearing — caught TD220 (stale TDD pseudocode) directly.
- Template comments double as inline examples and as "not applicable" fallback text for conditional sections.
- Two explicit checkpoints (scope + narrative review) caught a real scope issue (WF-001 missing 0.1.2) before it propagated.
- Auto-update of user-workflow-tracking.md "Integration Doc" column by the creation script removed a manual step.
- Customization guide's "completeness check" gave a concrete self-review bar.

### What could be improved
- Task definition's example invocation for `Update-TechDebt.ps1` uses non-existent parameters (`-Title`, `-Category`, `-Severity`); actual params are `-Description`, `-Dims`, `-Location`, `-Priority`, `-EstimatedEffort`.
- `New-IntegrationNarrative.ps1` can't find the Integration Narratives section in PD-documentation-map.md (regex matches `### Integration Narratives`, actual heading is `../../../../feedback-forms### \`technical/integration/\` — Integration Narratives`). Warning is shown but a successful run claims full automation.
- `New-IntegrationNarrative.ps1` post-run banner is noisy; the real warning about doc-map mismatch is buried in it.

### Specific suggestions
1. **Fix the Update-TechDebt.ps1 example in the task definition** — replace the bad invocation in Step 7 with one using the real parameters. Suggested:
   ```bash
   pwsh.exe -ExecutionPolicy Bypass -File process-framework/scripts/update/Update-TechDebt.ps1 -Add -Description "..." -Dims "DA" -Location "..." -Priority "Low" -EstimatedEffort "S (≤2h)" -Confirm:\$false
   ```
2. **Align New-IntegrationNarrative.ps1 section finder with actual doc-map heading** — either relax the regex to `### .* Integration Narratives` or add an alias. Alternatively, rename the doc-map subsection to the simpler form.
3. **On warning-path, print the exact markdown line that should be added manually** so the human just pastes it into the doc-map at the known location.
4. **Trim the success banner** in `New-IntegrationNarrative.ps1` output — the "customize the template" message is already in the task definition and template comments.
5. **Add a Step 11 sub-bullet** "verify PD-documentation-map.md contains the new entry; if script warned, add manually" so the finalization doesn't silently skip the map update.
