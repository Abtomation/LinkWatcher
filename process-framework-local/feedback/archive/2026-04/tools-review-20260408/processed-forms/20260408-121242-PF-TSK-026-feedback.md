Human partner provided 3 rounds of design feedback during concept review:

1. **Tracking location**: Integration Narratives should be tracked in `user-workflow-tracking.md`, not `feature-tracking.md` — narratives have a 1:1 relationship with workflows, not features. AI agent had incorrectly proposed feature-level tracking.
2. **Task ownership**: Implementation Finalization (feature-level) is the wrong owner. A new lightweight task is needed because Integration Narratives operate at the workflow level. The task must verify against source code (not just trust TDDs). Should be positioned before E2E test case creation as input.
3. **Maintenance scope**: Bug Fixing alone is insufficient — Code Refactoring and Feature Enhancement also need Integration Narrative maintenance checks. Documentation Alignment Validation must also validate narratives.
4. **Script auto-update**: `New-IntegrationNarrative.ps1` should auto-update `PD-documentation-map.md` (following `New-Task.ps1` pattern).
5. **Full Mode**: Corrected AI agent's incorrect Lightweight Mode assessment — the extension creates new file types and needs new templates/guides/scripts, which triggers Full Mode per PF-TSK-001 scope criteria.
6. **Column vs link**: Keep separate "Integration Doc" column in `user-workflow-tracking.md` for clarity (over linking from Workflow ID).
