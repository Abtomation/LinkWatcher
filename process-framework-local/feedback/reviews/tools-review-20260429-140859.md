## Cross-Group Themes

### Theme 1: Update-BugStatus.ps1 -RelatedFeature parameter missing (4/40 forms)

Surfaced in all 3 PF-TSK-041 (Bug Triage) forms and 1 PF-TSK-007 (Bug Fixing) form. Triage routinely corrects the Related Feature column during triage; without the parameter, agents must do a manual table-cell edit after every script invocation. **Filed as PF-IMP-621 (HIGH).**

### Theme 2: PF-TSK-005 mixes feature-review and bug-fix-review checklist (4/5 forms)

Steps 12 (review focus areas), 16-17 (perf/accessibility), and the State Tracking section assume feature reviews. For bug-fix reviews of CLI/backend code, many axes are inherently N/A. Multiple forms suggest Dims-gating per the bug's Dims column or a lightweight bug-fix-review path. **Filed as PF-IMP-622 (MEDIUM).**

### Theme 3: PF-TSK-009 Step 12 tool_doc_id naming convention undocumented (4/17 forms)

Task definitions use `PF-TSK-NNN`; templates/guides/scripts/context-maps use filename. Currently agents must fail-then-retry to discover the convention. PF-IMP-591 (now completed) added validation; this gap is the doc complement. **Filed as PF-IMP-623 (MEDIUM).**

### Theme 4: Update-TechDebt.ps1 -Dims help/validation mismatch (2/5 PF-TSK-005 forms)

Help text says "Space-separated if multiple" but `[ValidateSet]` enforces single value only. Cross-tool inconsistency too — bug-tracking accepts multi-value Dims. Cost retries every session. **Filed as PF-IMP-624 (MEDIUM).**

### Theme 5: Update-ProcessImprovement.ps1 status-edge-case ergonomics cluster (multiple PF-TSK-009 forms)

5 distinct ergonomic gaps surfaced across PF-TSK-009 forms, all small but compounding:
- Silent-drops `-ValidationNotes` for Deferred status (PF-IMP-625)
- Em-dash `-Impact` for Rejected undocumented (PF-IMP-626)
- Pipe-parser error needs `\|` / `&#124;` hint (PF-IMP-627)
- Bash-backtick truncation in `-ValidationNotes` (PF-IMP-628)
- "Row already in Completed" misdiagnoses as pipe issue (PF-IMP-629)

### Theme 6: Validate-StateTracking.ps1 surfaces are coupled (2 forms — PF-TSK-014)

Surface 6 (FeatureDeps) crash from PF-IMP-590 blocks surfaces 7-15, breaking PF-TSK-014 Lightweight's "0 errors across all surfaces" completion gate. Surface independence (continue-on-error) decouples this. **Filed as PF-IMP-640 (MEDIUM).** Also: 115 template-frontmatter-schema warnings drown signal — `-Quiet`/`-ErrorsOnly` mode (PF-IMP-646).

### Theme 7: Triple-🚨 success banner in `New-*.ps1` scripts (multiple forms across all groups)

Confirmed in `New-FeedbackForm.ps1`, `New-ProcessImprovement.ps1`, `New-Guide.ps1`, `New-FrameworkExtensionConcept.ps1`, `New-Handbook.ps1`. PF-IMP-584 fixed `New-IntegrationNarrative.ps1`; the shared pattern lives in `document-creation-script-template.ps1` Pattern 1. **Per user "observe before broadening" memory, broader sweep IMP deferred** until PF-IMP-584's narrow fix is observed in practice.
