## Cross-Group Themes

### Theme 1: Feedback-form template emits unused Tool stubs (4+ forms across 3 task groups)

`New-FeedbackForm.ps1` always emits 3-4 Tool skeletons regardless of actual tool count (including under SingleTool/TaskLevel types). Agents have to manually delete or rewrite unused sections, creating dead weight. Seen in PF-TSK-007, PF-TSK-009 (PF-FEE-972), and PF-TSK-030. **Fix**: `-ToolCount N` parameter → PF-IMP-582.

### Theme 2: Script success banners burying real warnings (5+ forms in PF-TSK-083)

`New-IntegrationNarrative.ps1` emits a triple-🚨 "CRITICAL: TEMPLATE CREATED - EXTENSIVE CUSTOMIZATION REQUIRED" block on every run. The message is already in the task definition and template header comment, so the third restatement is noise. Real warnings (section-not-found, ID collision) get buried. **Fix**: Trim banners → PF-IMP-584.

### Theme 3: Task-definition example drift from real script parameters

PF-IMP-335 previously fixed `Update-TechDebt.ps1` parameter names in 13 task files (`-Category` → `-Dims`), but PF-TSK-083 (newer) was missed and still uses non-existent `-Title`/`-Severity`/`-Category`. First invocation always fails until the agent reads `-?`. Suggests a periodic "example-vs-actual-params" validation pass could catch future drift. **Fix now**: PF-IMP-578.

### Theme 4: Silent-success failure modes

Both `New-IntegrationNarrative.ps1` (doc-map append, workflow-tracking update) and `feedback_db.py` (wrong `--tool` value) succeed visibly while failing functionally. Consistently flagged as the worst failure mode. Specific fixes: PF-IMP-586 (read-after-write in PF-TSK-083), PF-IMP-591 (`--list-tools` validation), PF-IMP-592 (print resolved DB path).

### Theme 5: Lightweight-path verification gates missing (PF-TSK-014, 4 of 6 forms)

The Lightweight path lacks an explicit grep/downstream-reference-sweep step before the execution checkpoint. Human partner repeatedly asked "did you grep check?" in sessions. Propagation of status labels, renames, etc. consistently hits 2-15+ unexpected files. **Fix**: PF-IMP-583.
