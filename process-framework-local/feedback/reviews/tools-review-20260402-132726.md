## Cross-Group Themes

### Theme 1: Partial Tracking Automation (~10/21 forms)
New-ValidationReport.ps1 updates only the reports registry row, leaving 5+ manual edits (progress matrix, metrics, change log, overall progress, next session) per session. This is the most frequently cited automation gap across all validation dimension tasks.

### Theme 2: LinkWatcher Race Condition on State Files (~8/21 forms)
Edit tool fails when LinkWatcher modifies the validation tracking file between read and edit operations. Workarounds (retry, sed, Python atomic writes) exist but add friction. Affects all validation sessions that update shared tracking files.

### Theme 3: Validation Report Template Boilerplate (~6/21 forms)
PF-TEM-034 includes appendices, methodology, and remediation sections that are empty or formulaic for clean-passing validations. Multiple forms suggest a "lite" variant or conditional sections.

### Theme 4: Tech Debt Quality Gate Misinterpretation (2/21 forms, user-reported)
AI agents treated the quality gate as a severity filter (skipping low-priority items) when it's a validity filter (preventing invalid items). User caught omissions and had them corrected in-session. High impact on process correctness.

### Theme 5: Update-TechDebt.ps1 Discoverability (3/21 forms)
`-Dims` parameter with 2-letter codes is not intuitive. Multiple attempts with wrong parameter names before finding the correct code. Quick fix via `-ListDims` switch.

### Theme 6: Documentation Drift from Enhancement/Bug Fix Tasks (2/21 forms)
TDDs and FDDs not updated when enhancements or bug fixes change public APIs. Root cause identified — but already addressed by completed PF-IMP-225 and IMP-050.
