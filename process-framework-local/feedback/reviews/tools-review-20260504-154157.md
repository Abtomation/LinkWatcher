## Cross-Group Themes

### Theme 1: Verbose-by-default automation scripts (8+ forms across PF-TSK-009 + PF-TSK-010)

`Update-ProcessImprovement.ps1`, `New-ProcessImprovement.ps1`, and `New-FeedbackForm.ps1` all emit ~9-12 lines of INFO/SUCCESS narration per invocation by default. Across PF-TSK-009 sessions (5-6 invocations each) and PF-TSK-010 (1 invocation each), accumulated noise is significant. PF-IMP-671 proved the default-quiet recipe on `Update-TechDebt.ps1`; sweep IMPs filed (PF-IMP-697, PF-IMP-700) for sibling scripts.

### Theme 2: Root-cause-vs-symptom application gap (3+ PF-TSK-009 forms)

Repeating pattern: agent proposes symptom-level fix at Step 6 (e.g., add a `-Quiet` flag, add a defensive `if`-check, add a documentation callout); human partner pushes back; root-cause fix turns out to be smaller and cleaner. The Minimum Viability criterion at Step 3 catches "smallest change" but doesn't distinguish "smallest symptom-level change" from "smallest root-cause change". PF-IMP-698 codifies the prompt.

### Theme 3: Archive concurrency between parallel Tools Review sessions (2 PF-TSK-010 forms)

Same-date Tools Review sessions both write to `tools-review-YYYYMMDD/processed-forms/`, causing visible mismatch in archive count. Recovery cost ~10-15 min per session; near-miss of erasing another session's correct archive. PF-IMP-699 proposes session-numbered subfolder OR explicit "combined-archive expected" guidance.

### Theme 4: Help-output discoverability gap (2+ forms)

`pwsh.exe -File path.ps1 -?` only renders SYNOPSIS+SYNTAX+DESCRIPTION; `.PARAMETER` and `.NOTES` blocks (where critical limits like the 500-char Description cap and BASH GOTCHA warnings live) require `Get-Help -Detailed`/`-Parameter <Name>`. Agents discover constraints only on validation failure. PF-IMP-703 addresses the New-ProcessImprovement.ps1 Description-length case specifically.

### Theme 5: Tracking-file documentation drift from upstream refactoring (2 forms across PF-TSK-022/PF-TSK-030)

PD-REF-220 left performance-test-tracking.md Tolerance column out of sync with the new code assertions; caught only by the next audit's Minor Fix Authority. Upstream cause is PF-TSK-022 not updating mirrored documentation; downstream catch is PF-TSK-030 not having a tracking-consistency sub-check. PF-IMP-710 adds the audit-side check.
