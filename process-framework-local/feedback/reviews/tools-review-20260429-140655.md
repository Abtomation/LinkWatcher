## Cross-Group Themes

### Theme 1: Run-Tests.ps1 output capture (10+/34 forms — 29%)

The wrapper's success indicator decouples from pytest's actual exit status, and the pytest summary line is missing from output. This single tool is the dominant friction point across PF-TSK-022 (visible in 10+ refactoring sessions). Workaround (direct pytest invocation) is straightforward but defeats the wrapper's purpose for the very step the parent task instructs you to run it for. Two distinct sub-issues: (a) summary passthrough (PF-IMP-647), (b) `-All` silently excluding `@pytest.mark.slow` (PF-IMP-648). Highest-leverage single set of fixes from session 2.

### Theme 2: Help-text discoverability for script constraints (4+/34 forms)

Multiple scripts have ValidateSet members and length constraints that are visible only at runtime when validation fails:
- New-BugReport.ps1: -DiscoveredBy ValidateSet (existing PF-IMP-223), -Environment ValidateSet, -Component/-Title length limits.
- New-ProcessImprovement.ps1: -Description 500-char limit (PF-IMP-659).
- Update-TechDebt.ps1: -PlanLink MSYS leading-slash hazard (PF-IMP-658).
- New-RefactoringPlan.ps1: -Lightweight/-DocumentationOnly/-Performance mutual exclusivity decision (PF-IMP-649).

Each costs 1-3 retry cycles per occurrence. Pattern is consistent enough to warrant a script-development convention: surface ValidateSet members + length constraints + path-mangling hazards in `-?` help by default.

### Theme 3: L8 Documentation & State Updates shortcut catalog gaps (5+/34 forms)

The existing Tier 1 / Test-only / Documentation-only shortcuts are widely praised but agents repeatedly construct ad-hoc batched-N/A justifications for cases the existing shortcuts don't cover:
- Framework-infrastructure refactoring (PF-IMP-653, 3+ forms).
- Internal-helper / byte-identical optimization (PF-IMP-654, 2 forms).
- Build-config-only changes (PF-IMP-668, 1 form).
- User-handbook check (PF-IMP-663, 1 form).
- Observability-only / logging-only (registered via PF-IMP-654; 1 form).

Pattern: the L8 shortcut catalog needs ongoing curation as new test-only-but-not-quite-test patterns emerge.

### Theme 4: Step 1 "independently verify TD descriptions" — universally praised (8+/27 PF-TSK-022 forms explicitly cite)

The single most valuable instruction in the Lightweight Path. Concrete catches across forms: TD220 (incorrect rationale claiming Phase 2 prefix-match handles parent refs implicitly), TD228 (orphan-table claim verification), TD234 (1-section→9-section scope expansion), TD244 (call-count off by 70%), TD247 (cores×100% scaling unit mismatch), TD248 (incoherent inter-iteration sleep "fix"), TD250 (verified prescription was sound). Worth highlighting as a deliberate framework feature rather than a generic warning.
