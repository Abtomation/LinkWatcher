# Structure Change Proposal Template

## Overview
Move linkwatcher/ package directory to src/linkwatcher/ to adopt standard Python src layout, resolving naming conflict with runtime directory

**Structure Change ID:** SC-PENDING
**Proposer:** AI Agent & Human Partner
**Date Proposed:** 2026-04-16
**Target Implementation Date:** 2026-04-16

## Current Structure

The Python source code package sits at the project root alongside the runtime directory:

```
LinkWatcher/                    (project root)
├── linkwatcher/                ← source code package
│   ├── __init__.py
│   ├── service.py
│   ├── database.py
│   ├── parsers/
│   └── ... (16 modules + 1 subpackage)
├── LinkWatcher_run/            ← runtime directory (workaround name)
│   ├── start_linkwatcher_background.ps1
│   ├── setup_project.py
│   └── ...
├── main.py
├── pyproject.toml
└── ...
```

**Problem**: In other projects where LinkWatcher is installed globally, the runtime directory is named `src/linkwatcher`. In this project, that name is already taken by the source code package, forcing the workaround name `LinkWatcher_run/`.

## Proposed Structure

Adopt the standard Python `src` layout — the source code moves one level deeper:

```
LinkWatcher/                    (project root)
├── src/
│   └── linkwatcher/            ← source code package (unchanged name)
│       ├── __init__.py
│       ├── service.py
│       ├── database.py
│       ├── parsers/
│       └── ...
├── LinkWatcher_run/            ← runtime directory (rename to linkwatcher/ is now possible)
├── main.py
├── pyproject.toml
└── ...
```

**Key**: The package name stays `linkwatcher`. All Python imports (`from linkwatcher.service import ...`) remain unchanged. Only file path references change (`src/linkwatcher/service.py` → `src/linkwatcher/service.py`).

## Rationale

### Benefits
- **Resolves naming conflict** — frees the `src/linkwatcher` name for the runtime directory, enabling consistent naming across all projects
- **Standard Python convention** — `src` layout is the recommended project structure for Python packages (PEP 517/518 ecosystem)
- **Cleaner separation** — source code clearly separated from runtime files, tests, docs, and tooling
- **No import changes** — package name preserved; all `from linkwatcher...` imports stay the same

### Challenges
- **~100 files with path references** — all documentation referencing `src/linkwatcher/service.py` etc. needs updating (handled by LinkWatcher automatically)
- **6 files need manual edits** — `pyproject.toml`, `dev.bat`, `deployment/install_global.py`, `.claude/settings.local.json`, `.linkwatcher-ignore`
- **Reinstall required** — `pip install -e .` must be re-run after the move
- **`egg-info` regeneration** — stale `linkwatcher.egg-info/` must be deleted and regenerated

## Migration Strategy

### Phase 1: Preparation
- Stop LinkWatcher to prevent it from processing the move in real-time while we're still preparing
- Delete stale `linkwatcher.egg-info/` directory
- Verify `src/` directory exists (already present, empty)

### Phase 2: Directory Move
- Move `src/linkwatcher` → `src/linkwatcher/` (single operation)
- Restart LinkWatcher and wait for it to process all path reference updates
- Verify via grep that no stale `src/linkwatcher` path references remain (excluding `src/linkwatcher/`)

### Phase 3: Manual Edits
- Update `pyproject.toml` packaging configuration
- Update `dev.bat` linting/formatting paths
- Update `deployment/install_global.py` copy source path
- Update `.claude/settings.local.json` path
- Update `.linkwatcher-ignore` suppression rule

### Phase 4: Verification
- Run `pip install -e .` to reinstall package with new layout
- Run full test suite (`pytest test/automated/`)
- Run `python main.py --help` to verify CLI works
- Run link validation (`python main.py --validate`) to check for broken references

## Testing Approach

- All 247+ existing test methods pass after the move

### Success Criteria

- LinkWatcher starts and runs normally after the change

## Rollback Plan

### Trigger Conditions
- Test suite failures that cannot be resolved by config fixes
- LinkWatcher unable to process the move correctly (mass broken references)

### Rollback Steps
1. Move `src/linkwatcher/` back to `src/linkwatcher` (LinkWatcher will reverse-update all references)
2. Revert `pyproject.toml` to `where = ["."]`
3. Revert `dev.bat`, `deployment/install_global.py`, `.claude/settings.local.json`, `.linkwatcher-ignore`
4. Re-run `pip install -e .`
5. Verify test suite passes
