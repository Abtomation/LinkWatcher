# E2E Acceptance Testing

This directory contains the end-to-end acceptance testing infrastructure for validating system behavior through human interaction with the running application, organized per user workflow.

## Directory Structure

```
e2e-acceptance-testing/
├── README.md                              # This file
├── <workflow-slug>/                       # One subdir per workflow (WF-NNN in user-workflow-tracking.md)
│   ├── templates/                         # PRISTINE test fixtures (NEVER modified during tests)
│   │   ├── master-test-<workflow>.md      # TE-E2G master test — quick validation sequence for the workflow
│   │   └── E2E-NNN-<name>/                # Individual TE-E2E test case
│   │       ├── test-case.md               # Steps, preconditions, expected results
│   │       ├── project/                   # Starting state files
│   │       └── expected/                  # Post-test expected state
│   ├── workspace/                         # GENERATED by Setup-TestEnvironment.ps1 (gitignored)
│   └── results/                           # Test session logs (gitignored)
```

Per-workflow directories are scaffolded automatically when a workflow is added via `New-WorkflowEntry.ps1`, which chains into `New-TestInfrastructure.ps1 -Update`.

## How to Use

### 1. Set Up Environment

```bash
pwsh.exe -ExecutionPolicy Bypass -File process-framework/scripts/test/e2e-acceptance-testing/Setup-TestEnvironment.ps1 -Workflow "<workflow-slug>" -Clean
```

### 2. Execute Tests

Follow the master test's Quick Validation Sequence first. If it fails, run individual test cases to isolate.

### 3. Verify Results

```bash
pwsh.exe -ExecutionPolicy Bypass -File process-framework/scripts/test/e2e-acceptance-testing/Verify-TestResult.ps1 -Workflow "<workflow-slug>" -Detailed
```

### 4. Record Results

```bash
pwsh.exe -ExecutionPolicy Bypass -File process-framework/scripts/test/e2e-acceptance-testing/Update-TestExecutionStatus.ps1 -Workflow "<workflow-slug>" -Status "Passed"
```
