# Manual Testing Framework

This directory contains the manual testing infrastructure for validating system behavior through human interaction with the running application.

## Directory Structure

```
manual-testing/
├── README.md              # This file
├── templates/             # PRISTINE test fixtures (NEVER modified during tests)
│   └── <group-name>/     # Test group (e.g., "basic-file-operations")
│       ├── master-test-<group-name>.md   # Quick validation sequence for the group
│       └── MT-NNN-<name>/               # Individual test case
│           ├── test-case.md             # Steps, preconditions, expected results
│           ├── project/                 # Starting state files
│           └── expected/                # Post-test expected state
├── workspace/             # GENERATED by Setup-TestEnvironment.ps1 (gitignored)
└── results/               # Test session logs (gitignored)
```

## How to Use

### 1. Set Up Environment

```bash
cd doc/process-framework/scripts/testing
pwsh.exe -ExecutionPolicy Bypass -Command '& .\Setup-TestEnvironment.ps1 -Group "group-name" -Clean'
```

### 2. Execute Tests

Follow the master test's Quick Validation Sequence first. If it fails, run individual test cases to isolate.

### 3. Verify Results

```bash
pwsh.exe -ExecutionPolicy Bypass -Command '& .\Verify-TestResult.ps1 -Group "group-name" -Detailed'
```

### 4. Record Results

```bash
pwsh.exe -ExecutionPolicy Bypass -Command '& .\Update-TestExecutionStatus.ps1 -Group "group-name" -Status "Passed"'
```
