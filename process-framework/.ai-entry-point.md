# AI Entry Point

> **🚨 This file is an ACTIVE STARTUP PROCEDURE — not passive context.** Execute each step below at the start of every session before responding to the user.

## Startup Procedure

Run through these steps in order at the start of every session:

1. **🔗 Start LinkWatcher**: `LinkWatcher_run/start_linkwatcher_background.ps1`
2. **⏱️ Start time tracking**: Call `get_current_time` (MCP server if available) to record session start
3. **🚨 Select a task**: Read the user's request, then select the appropriate task from [AI Task-Based Development System](ai-tasks.md). **This must happen before any code exploration, file reading, or work.** If no task fits, ask the human partner before proceeding.
4. **📋 Read the COMPLETE task definition** including its completion checklist before starting work

## General Instructions

- **🚨 CRITICAL**: Every task has a mandatory completion checklist including feedback forms — tasks are NOT complete until all checklist items are finished
- **⏱️ DURATION CALCULATION**: When creating feedback forms, calculate session duration from start time and manually enter it in the feedback form template
- **📊 CONTEXT MANAGEMENT**: Always refer to the Context Map linked in each task's Context Requirements section to understand component relationships
- **🔍 VISUAL NOTATION**: Use the [Visual Notation Guide](/process-framework/guides/support/visual-notation-guide.md) to interpret all diagram formats
- **🔗 LINK MAINTENANCE**:
  - LinkWatcher runs in background and automatically maintains all document references in real-time
  - Use any method to move/rename files (VS Code, File Explorer, git commands) — LinkWatcher handles all updates
  - **Check broken links**: Run `python main.py --validate` to scan the workspace for broken file references (results saved to `LinkWatcherBrokenLinks.txt`)
- **⚙️ POWERSHELL SCRIPT EXECUTION**:
  - **Preferred**: `pwsh.exe -ExecutionPolicy Bypass -Command '& .\Script.ps1 -Param "value" -Confirm:$false'` — wrap entire `-Command` in bash single quotes
  - **Fallback** (complex quoting): Write a temp `.ps1` file, execute with `-File`, then clean up
  - See [Script Development Quick Reference](/process-framework/guides/support/script-development-quick-reference.md#powershell-script-output-not-visible-ai-agents) for details

## Purpose

This file is designed to be detected early in an AI agent's session to ensure the agent understands the task-based development approach used in this project before reading unnecessary files.
